# Web Store screenshots — capture guide (owner)

Claude Code can't take live Gmail screenshots, so these are captured by the
owner from real usage. The Chrome Web Store accepts **1–5 screenshots** at
**1280×800** or **640×400** (1280×800 strongly preferred — sharper in the
listing). PNG or JPEG. Capture on a clean Gmail account with no sensitive
content visible.

Tips:
- Use a real but disposable recipient (e.g. your own alias) so no private
  email addresses leak into the store listing.
- Capture at 1280×800 exactly (or crop to it). On a Retina Mac, a 640×400
  region captured at 2× = 1280×800.
- Keep the new teal UI visible — these double as the "after the rebrand" proof.

## The 5 to capture

1. **Schedule Send modal with Optimize-for-X open.** Compose an email, click
   the (relabelled) Schedule send, and open the modal with the Optimize
   section engaged and a recipient selected. The hero shot — shows the core
   feature. Make sure the teal Schedule button + brand mark show.

2. **Onboarding Step 1 (Welcome).** The first-run screen with the new symbol
   above the headline and the consent checkbox. Shows the brand + the
   privacy-first posture.

3. **Settings → Pinned Timezones.** The Settings page on the Pinned Timezones
   section (teal sidebar active item, teal chips). Shows configurability.

4. **Outside-hours soft warning (§5.5.1).** Trigger a regular Send outside
   your configured working hours so the 3-choice soft-warning modal appears
   (Proceed / Reschedule / Cancel). Shows the "protects your professional
   brand" angle.

5. **Privacy & Data (Settings).** The Privacy & Data section showing
   Export / Delete — the "your data, your control, on-device" surface.
   Reinforces the zero-server story (matches the clean security audit).

After capturing, drop the PNGs in this folder (e.g. `01-schedule-modal.png`)
and use them in the Web Store listing flow.
